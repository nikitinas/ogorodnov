## Портал документации методики Огороднова

Проект — это веб-сайт на [Next.js](https://nextjs.org), который собирает в единый интерфейс все материалы из каталога `../docs`. Контент автоматически индексируется из Markdown-файлов и отображается с удобной навигацией, хлебными крошками и тематическими подборками.

### Основные возможности

- автогенерация дерева разделов по структуре файловой системы `docs/`;
- статическая генерация страниц для каждого Markdown-документа;
- рендеринг Markdown с поддержкой таблиц и списков (`remark-gfm`);
- домашняя страница с выделением ключевых разделов: методика, приложения, веб-платформа;
- адаптивный интерфейс с «липким» меню и стилями для чтения.

### Старт проекта

```bash
cd website
npm install
npm run dev
```

По умолчанию сайт доступен на [http://localhost:3000](http://localhost:3000).

### Полезные команды

- `npm run dev` — запуск локального дев-сервера.
- `npm run lint` — проверка линтером (ESLint + TypeScript).
- `npm run build` — production-сборка.
- `npm run start` — запуск собранного приложения.

### Организация контента

Папка `docs/` расположена на уровень выше проекта и содержит тематические блоки:

- `docs/method` — основы методики, источники, обзор;
- `docs/apps` — концепции приложений и геймифицированных тренажёров;
- `docs/website` — структура сайта, аудитории, дизайн-концепция.

Каждый Markdown-файл автоматически получает slug (на основе названия файла) и превращается в страницу. Нагрузке подлежат только `.md` файлы; остальные ресурсы можно подключать вручную через `public/`, если потребуется отображение.

### Развёртывание

Проект готов к деплою на любую платформу, поддерживающую Next.js (например, Vercel). Перед деплоем убедитесь, что папка `docs/` доступна в файловой системе окружения и расположена рядом с корнем приложения.
